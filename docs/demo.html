How to Set Up a Basic Vue Chrome Extension

This tutorial walks you through building the simplest possible Chrome Extension using Vite + Vue. Each step explains what to do and why you are doing it, so even a new student can follow without confusion.

STEP 0 — Create a new folder

Why this step? We start by preparing a clean workspace so all files stay organized.

mkdir vue-extension
            cd vue-extension

This folder will hold the project source code and the built Chrome Extension.

STEP 1 — Create a fresh Vite + Vue project

Why this step? Vite gives us a fast development environment. Vue lets us build a modern UI for the popup.

npm create vite@latest . -- --template vue
            npm install

This creates the standard Vue folder structure with src/, index.html, and Vite configuration.

STEP 2 — Create popup.html (root level)

Why this step? Chrome Extensions need a physical HTML file for the popup UI. Vue alone is not enough; Chrome does not load Vite dev pages.

Create a new file at the project root:

popup.html

Paste this inside:

<!DOCTYPE html>
            <html>
              <head>
                <meta charset="UTF-8" />
                <title>Vue Popup</title>
              </head>
              <body>
                <div id="app"></div>
                <script type="module" src="./src/popup.js"></script>
              </body>
            </html>

This loads the Vue app inside Chrome’s popup window.

STEP 3 — Create popup.js

Why this step? Chrome cannot execute .vue files directly. We need a JavaScript entry point that mounts Vue into the popup.

Create:

src/popup.js

Paste:

import { createApp } from "vue";
            import App from "./App.vue";
            
            createApp(App).mount("#app");

This tells Chrome: "Insert the Vue application into popup.html."

STEP 4 — Replace App.vue with a simple counter

Why this step? We want the simplest possible test to verify that the Vue app actually renders inside Chrome’s popup.

Open:

src/App.vue

Replace all contents with:

<script setup>
            import { ref } from "vue";
            const count = ref(0);
            </script>
            
            <template>
              <h2>Vue Extension Counter</h2>
              <button @click="count++">Count: {{ count }}</button>
            </template>

If the counter works in Chrome, the extension setup is correct.

STEP 5 — Create manifest.json

Why this step? Every Chrome Extension needs a manifest to tell Chrome what files to load and what behavior is allowed.

Create this file:

public/manifest.json

Paste:

{
              "manifest_version": 3,
              "name": "Vue Test Extension",
              "version": "1.0.0",
              "action": {
                "default_popup": "popup.html"
              }
            }

This tells Chrome: “Use popup.html as the popup UI.”

STEP 6 — Create a working Vite config

Why this step? By default, Vite does not build Chrome extension entry points. We must tell it:
✔ Build popup.html
✔ Build popup.js
✔ Output a clean dist/ folder Chrome can load

Edit:

vite.config.js

Replace with:

import { defineConfig } from "vite";
            import vue from "@vitejs/plugin-vue";
            import { resolve } from "path";
            
            export default defineConfig({
              plugins: [vue()],
              build: {
                outDir: "dist",
                rollupOptions: {
                  input: {
                    popup: resolve("popup.html"),
                  }
                }
              }
            });

This ensures popup.html and its scripts are built correctly.

STEP 7 — Build the project

Why this step? Chrome cannot interpret Vue or Vite source files. Everything must be transformed into plain JavaScript inside a dist/ folder.

npm run build

This generates a folder called dist/ containing the ready-to-install extension.

STEP 8 — Load the extension into Chrome

Why this step? Chrome Extensions must be loaded manually while developing them.

  1. Open chrome://extensions
  2. Enable Developer Mode
  3. Click Load Unpacked
  4. Select your dist/ folder

If you click the extension icon and see the counter working, everything is set up correctly.
